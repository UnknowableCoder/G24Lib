Detailed Description

template<class Type, class indexer = g24_lib::Definitions::default_unsigned_indexer, class memory_manager = default_memory_manager<Type, indexer>, bool default_is_copy = true>
class g24_lib::simple_array< Type, indexer, memory_manager, default_is_copy >

A container that holds an array of objects in a given memory context.

Template Parameters
TypeThe type of the objects in the array.
indexerA type that is appropriate to hold the size of the array.
memory_managerThe memory manager corresponding to the memory context where the array will be held.
default_is_copyDetermines the behaviour of copy constructors and assignment operators: if false, instead of copying the elements into a new array, only the pointer is copied (and the memory will not be deleted upon destruction). In essence, if true, the array acts as something akin to an std::unique_ptr, while, if false, it is more like an std::weak_ptr. (This is currently only used to pass arrays into CUDA kernels. For the end user, default_is_copy should be true.)
Remarks
Even if default_is_copy is false, this will still act as an std::unique_ptr if it isn't copy-constructed and/or assigned from another g24_lib::simple_array with the same memory_manager.
Just to be extra clear, for memory validity to be ensured, if the memory contexts do not match, the g24_lib::simple_array will still act as an std::unique_ptr, allocating the appropriate amount of memory and being able to delete it upon destruction.
Warning
When acting as a raw pointer, the memory may become invalid if the object from which that pointer was copied is destructed or resized.
Precondition
indexer must satisfy the constraints placed on the choice of g24_lib::Definitions::default_unsigned_indexer.
memory_manager must be a valid memory manager, that is, g24_lib::is_memory_manager<memory_manager> should be true.
Warning
Due to the possible need to copy objects between different memory contexts, namely when interfacing with CUDA-enabled GPUs, copy constructors or assignment operators of Type will not be invoked during these transfers: Type will only be shallow-copied (that is, operations akin to std::memcpy that copy bytes). For most numerical applications, this shouln't pose too much of a problem, but it is a limitation one must take into account carefully.
Todo:
Try to accomodate copy constructors/assignment operators into this.

Parameters
fThe function to apply on the elements of the array.
argsExtra arguments to be passed to f
Template Parameters
FuncThe type that holds the function to be applied to the elements of the array.
ArgsThe types for any extra arguments to Func.
Precondition

Remarks
