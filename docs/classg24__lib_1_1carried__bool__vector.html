Detailed Description

template<class Block, class indexer = g24_lib::Definitions::default_unsigned_indexer, class memory_manager = default_memory_manager<Block, indexer>, bool default_is_copy = true>
class g24_lib::carried_bool_vector< Block, indexer, memory_manager, default_is_copy >

A container that holds an array of booleans in a given memory context in a space efficient way, by packing them as the binary representation of another object.

Template Parameters
BlockThe type of the objects whose binary representation will hold the booleans (the blocks).
indexerA type that is appropriate to hold the size of the array.
memory_managerThe memory manager corresponding to the memory context where the array will be held.
default_is_copySee g24_lib::simple_array.
Precondition
Block must support bitwise operations and should not have padding (that is, all bytes must be part of the object's data). In essence, one should use something that behaves like an integer type.
indexer must satisfy the constraints placed on the choice of g24_lib::Definitions::default_unsigned_indexer.
memory_manager must be a valid memory manager, that is, g24_lib::is_memory_manager<memory_manager> should be true.
Remarks
Though most mainstream implementations of std::vector<bool> are a template specialization that packs the booleans in a way quite similar to what's achieved here, it is not mandated by the standard to work that way. There are some trade-offs since accessing a bool requires bitwise operations
Todo:
Implement generalized construction from any form of container that returns booleans with operator[].

template<class Block , class indexer = g24_lib::Definitions::default_unsigned_indexer, class memory_manager = default_memory_manager<Block, indexer>, bool default_is_copy = true>
template<class parallel , class Func , class ... Args>
void g24_lib::carried_bool_vector< Block, indexer, memory_manager, default_is_copy >::operate_on_all ( Func &&  f,
Args &&...  args 
)
inline

Performs operation given by f on every element of the vector.

Parameters
fThe function to apply on the elements of the vector.
argsExtra arguments to be passed to f
Template Parameters
FuncThe type that holds the function to be applied to the elements of the vector.
ArgsThe types for any extra arguments to Func.
Precondition
f must take as arguments a reference to the vector and the index at which we're evaluating and any extra arguments passed to this function in args.
parallel must be a valid type of parallelism, that is, it must satisfy g24_lib::is_parallelism.
Remarks
This is constructed in such a way that all the vector accesses are thread-safe (as long as only the element of the provided index is written to). Effectively, the operations are parallelized over the array of Blocks, so there will be roughly v.size/v::chunk operations in parallel, each comprised of chunk function evaluations. For sufficiently complicated functions, it could prove more useful to operate on a g24_lib::simple_array of booleans and then convert it to a g24_lib::carried_bool_vector, as long as there is sufficient memory for it, of course.

)
inline

Sets the boolean with index i to the value of b.

Warning
This is not safe for multithreaded operations unless different blocks are being accessed.
Remarks
The access will surely be thread safe if the difference of the indices of the accessed booleans is greater than chunk, though particular cases can be constructed where a smaller difference still allows for thread safety. (Edge case: accessing the last boolean in a block and the first boolean in the next will be perfectly thread-safe.)
